# Discover Your Direction：设计说明

日期：2026-06-23

## 1. 产品定义

`discover-your-direction` 是一个中文优先的互动式 Codex Skill，服务于“已经积累了工作、兴趣、项目或人生经历，却说不清个人下一步方向”的用户。内容或商业方向只有在它源于这些个人积累时才属于范围。

它不承诺在 15 分钟内发现终身使命。它帮助用户从个人证据中提出三种“可能的自我”，选择一项低成本、可证伪的身份实验，并通过现实反馈减少不确定性。

核心结果不是一份漂亮定位，而是一项用户愿意执行、能够带来决定性新信息的实验。

### 可检验的问题定义

对于一名有真实经历但说不清下一步方向的用户，这个 Skill 能否在约十五分钟的核心对话中：

1. 从自述与可选资料中生成三个实质不同、可追溯证据的方向假设；
2. 帮助用户选择一个七天内可执行、具有明确推翻条件的身份实验；
3. 让现实反馈足以增强、削弱或重新定义至少一项关键假设；
4. 全程不把 AI 推测包装成事实或终身结论。

Skill 是否成功，不取决于首个方向最终成立，而取决于用户是否获得了能改变下一步判断的证据。

## 2. 设计原则

1. **方向是实验，不是裁决。** 所有方向均为暂时假设，必须包含未知项和推翻条件。
2. **行动证据优先于自我标签。** 优先分析具体事件、过去行为、真实结果和他人实际投入的时间、金钱、声誉或其他承诺。
3. **过去证据不限制未来身份。** 保留缺少既有证明、但值得低成本试演的新方向。
4. **发散与收敛分离。** 前十分钟探索，后五分钟选择，不在探索阶段评分。
5. **不使用伪精确总分。** 只标记证据来源、强弱与冲突，不计算综合分数。
6. **验证方式服从方向。** 内容只是媒介之一，也可使用咨询、服务、工具、工作坊或帮助真人进行验证。
7. **坏消息具有价值。** 优先选择能以最低成本获得决定性信息的实验。
8. **用户拥有解释权。** AI 必须允许用户纠正材料解读，不把归纳结果冒充事实。

## 3. 范围

### 首版包含

- 一次只问一题的中文互动访谈。
- 六个核心问题，最多三次自适应追问。
- 可选的本地资料与 AI Agent 对话镜像。
- 三个精简方向假设及证据说明。
- 一项主实验的选择。
- 按需生成七天实验设计与对应执行材料。
- 隐私边界、证据溯源、低置信度与失败处理。

### 首版不包含

- 职业测评、人格测评或心理诊断。
- 保证找到使命、利基或商业成功。
- 战术内容日历、普通商业验证、泛化优势清单，以及与个人积累无关的小实验。
- 由 AI 发布内容、联系他人、发送材料、招募参与者或交付实验；这些动作始终由用户本人完成。AI 也不在未经明确授权时修改用户资料。
- 用点赞或用户的主观喜爱作为唯一成功指标。
- 在没有用户确认的情况下扫描整个工作区。

## 4. 总体流程

```text
按用户当前状态路由
├─ 尚无方向：可选资料镜像 → 证据访谈与框架校验 → 生成并选择方向
├─ 已选方向/已有实验：概括已知状态并确认一次 → 安全门槛与预注册 → 剩余执行
└─ 已有结果：直接复盘 → 继续、调整或停止
```

不得强迫用户从资料镜像或访谈起点重来。路由前先执行两条跨分支不变量：任何阶段只要涉及用户文件（包括文件中的返回结果或既有实验草稿），读取前都必须完整读取 `evidence-mirror.md`，解析并重述精确只读范围，取得明确确认；任何结果复盘，即使不涉及文件，也必须先加载并应用规范证据记录 schema 再分类观察。

已有结果但没有预注册时，只做谨慎的事后复盘：不得补造推翻条件，不得把事后解释伪装为预注册，也不得夸大证据。这些观察可以帮助形成暂定的下一项假设或现实决策，但不算预注册实验的更新，也不得称为证实或证伪了某项预注册实验。实验已开始但尚未结束时，预注册只约束后续行动，已完成部分仍按事后观察处理。

十五分钟是核心访谈与方向选择的体验目标，不是强制倒计时。用户需要展开重要事件时，应优先保证证据质量。资料数量较多时，读取和分析时间不计入十五分钟，并提前向用户说明。

## 5. 可选资料镜像

### 授权流程

1. 询问用户是否启用资料镜像；跳过后仍可完成全部访谈。
2. 只接受用户明确指定的文件或文件夹。
3. 解析出确切文件后，重述完整路径清单，说明只读，并要求用户明确确认后再读取。
4. 未获得新的同意，不跟随清单之外的链接、嵌入、附件、符号链接或引用。
5. 遇到密钥或凭据、系统文件、可执行文件或二进制、意外过宽的目录、无关个人数据时暂停，并说明对应的安全、格式、范围或隐私风险；对宽泛、可疑、敏感或过大的选择要求缩小范围。
6. 完成后列出实际使用的材料，只陈述 Agent 将其用于读取且没有修改；不声称能防止其他进程改动文件。

### 提取对象

- 反复出现的具体动作与问题类型。
- 别人重复寻求的帮助及其投入的时间、金钱、声誉或再次求助。
- 困难成果、可迁移能力和可能复利的职业资本。
- 自发坚持、能量变化和反复返回的主题。
- 不同资料之间的矛盾和反例。
- 用户在不确定情境中的观察、调整与判断。

### 规范证据记录

`evidence-mirror.md` 是唯一规范 schema。来源观察与当前 AI 推断使用两种分离且关联的记录：

- **来源观察记录**：包含 ID、`claim status=source observation/paraphrase`、观察内容、引用/转述、provenance、evidence kind 与 source reference。三轴分类只适用于来源观察。
- **来源引用（source reference）**：文件资料使用“文件路径 + 精确标题/行/消息/时间定位”；实时访谈使用“`current conversation` + 回合/消息定位”；已存聊天使用“转录文件 + 说话者/消息或时间定位”。
- **当前 AI 推断记录**：不是证据；包含 ID、`claim status=current AI inference`、关联的来源观察 ID、解释、替代解释、最强矛盾、缺失证据和 `strong`/`mixed`/`weak` 置信度；不得含 provenance、evidence kind 或 source reference。

用户描述的行动仍属于 `user self-report`，除非被独立来源佐证；模型回合只能作为 AI 对话线索。置信度由关联观察的具体程度、质量、独立重复和矛盾共同决定，不能只按类别判断；不得计算综合分数。

镜像最多输出五个暂定模式。每个模式列出关联的来源观察 ID 与 source reference；存在反例时必须单列最强已知矛盾或反例，并另列关键缺失证据。邀请用户纠正，且用户修正优先于 AI 解释。

## 6. 十五分钟核心访谈

### 对话规则

- 一次只问一个问题。
- 使用自然对话，不展示问卷进度压力。
- 六个核心问题最多追加三次追问。
- 追问优先要求真实事件、具体动作、相关人物和结果。
- 用户没有答案时允许跳过，不强迫制造故事。
- 六问是必须覆盖的证据维度，不要求机械重复资料中已经充分回答的问题。资料已有明确证据时，先概括并请用户确认；把有限追问留给冲突、缺口与反例。

### 六个核心问题

1. **关键事件回放**：选择一次做成、做砸或被反复求助的事件，依次还原情境、意外、行动、调整与反馈。
2. **真实需求、相关性或采用**：最近谁因为什么具体问题找过你、使用或复用了你的工作，或投入时间关注与回应？对方实际投入了什么，并在事后做了什么？
3. **职业资本**：过去几年做成过哪些困难的事？其中哪些能力可迁移、难复制或能继续复利？
4. **自发投入与能量**：什么事情无人要求你仍会持续琢磨？哪些事情做完后反而更有能量，并愿意在受挫后返回？
5. **反向经验**：哪条主流建议曾让你付出代价？什么具体经历迫使你改变判断？
6. **具体人群与未说真话**：哪类人正在反复遭遇什么问题？这个领域里大家在回避什么，而你有什么亲身证据？

### 框架校验

在前十分钟阶段结束前，AI 对最关键的一项事件给出两种合理但不同的解释，并询问：

- 哪一种更接近真实？
- 哪一处理解失真？
- 遗漏了什么难以描述但重要的能力？

用户的修正优先于 AI 的原始归纳。

## 7. 三种可能的自我

AI 生成三个相互有差异的方向假设。至少一个方向可以是证据尚弱但值得试演的新身份，避免所有建议都只是过去经历的延长线。

每张卡只包含五项：

1. **角色与对象**：以什么角色，为哪类受益者、利益相关者、同行、实践社群或有后果的工作场景，解决什么问题或推进什么实践。
2. **支持证据**：引用 `evidence-mirror.md` 的规范记录，不在此复制 schema；来源观察与 AI 解释分开，能力，需求、相关性或采用，以及能量分别标明强、混合或弱。
3. **最大未知**：当前最重要的证据缺口。
4. **推翻条件**：出现什么结果应降低对受检验假设组件的信心，而不是否定整个可能的自我。
5. **最小身份试演**：一次低成本、接近真实工作的行动。

默认每项只用一个短要点呈现，用户要求时再提供完整证据账本。不提供综合分数、排名或赢家；必须解释不同方向的不确定性，不把不可通约的证据强行相加。

## 8. 选择原则

用户从三个方向中选择一项主实验，另外两项保留在当前对话中，不宣布其错误；只有用户明确要求时才持久化保存。

选择顺序为：

1. 用户是否愿意真实进入该角色，而不只是喜欢它的描述。
2. 实验能否低成本获得决定性新信息或坏消息。
3. 是否同时触及真实受益者、利益相关者、同行、实践社群或有后果的工作场景，以及真实问题/实践和真实工作方式；不得结构性排除研究、手艺、基础设施或艺术方向。
4. 是否有机会发展稀缺能力或形成长期复利。

如果用户只按“最像自己”或“听起来最宏大”选择，Skill 应指出偏差并帮助比较实验的信息价值。

## 9. 七天实验设计

用户选择方向后，Skill 再按需进入第二阶段，不把完整实验包塞进十五分钟流程。

### 实验最低要求

- 由用户本人完成所有外联和交付。AI 只能起草材料；在本流程中绝不代为联系、发送、发布、招募或交付，这些动作始终由用户完成。
- 对两名正在从事相同或相近实践的人各计划一次尊重的外联尝试；完成对话是更强证据，但不是实验有效的必要条件。外联不安全、不合适或不切实际时，一场相关公开演讲、一份已发布案例研究或一次有监督的观察可替代一次外联；记录替代了哪次及原因，若两次均受同一限制，可使用两次替代。
- 仅在安全、知情同意且用户能力范围内，尝试帮助一名真实受益者、利益相关者、同行或实践社群处理具体问题；否则在忠实的、有后果的工作场景完成一小段实践。
- 根据方向选择合适媒介：内容、咨询、服务、小工具、工作坊或其他最小实践。
- 执行前预注册：被检验的需求、相关性或采用，以及能力、能量或角色适配假设组件；同时定义最大未知、动作、对象/场景、七天观察窗口、有效测试条件（目标匹配、曝光次数及分母、媒介、交付忠实度）、反证观察，以及不同结果如何更新对应组件并导向继续/调整/停止。
- 运行期间冻结解释规则，偏差另行记录，不得看到结果后静默改写成功标准。

### 安全与外联门槛

- 参与必须尊重且自愿，说明实验目的；尊重拒绝或不回复，未经邀请最多礼貌跟进一次。
- 数据最小化并去标识；保留引语或可识别细节前取得同意。“精确承诺”指精确记录观察到的行为，不是逐字保存个人细节。
- 工作必须小、边界明确、可逆，且在用户能力范围内。
- 医疗、法律、金融、治疗、safeguarding 或脆弱人群情境，不得用无资质建议或服务做实验；改用公开资料研究、与合格实践者的信息访谈或合格监督。涉及受监管场景或脆弱人群的观察必须是公开观察，或者同时具备适当同意与机构/专业批准；仅仅“非敏感”并不足够。

### 信号与承诺记录

不设通用阶梯；按角色和场景校准信号，并记录每个实质信号，而不是只报告最高信号：观察到的行为，时间/金钱/声誉/隐私/努力成本，目标匹配，熟人/陌生人/其他场景，次数，以及曝光分母。付费是可选且依角色而定。

以下只是一种常见商业场景的非通用“承诺阶梯”示例：

```text
礼貌回应或点赞
< 主动追问或提供详细经历
< 提交资料或转介绍
< 预约、持续试用或投入协作时间
< 付费或重复使用
```

熟人鼓励与陌生目标用户的主动承诺必须分开记录。赞美和未来承诺若无相关的高成本行为，不算承诺。

### 内部学习指标

- 是否愿意在受挫后再次进入该角色。
- 自主感是否增加。
- 是否形成新的可迁移能力。
- 是否产生可见进步或作品质量提升。
- 哪个意外迫使用户重新定义问题。

持续能量下降可以触发调整，不把坚持本身道德化。

### 决策规则

- **继续**：综合角色校准后的外部响应、忠实执行、内部体验、技能与学习，受检验组件获得支持。
- **调整**：执行产生了信息，但对象、角色、媒介、交付方式或测试设计需要改变。
- **停止**：预注册的关键组件被推翻，或用户真实体验不匹配。

一次七天实验只更新预注册的需求、相关性或采用，以及能力、能量或角色适配组件，绝不验证或否定整个身份。三十至九十天的小项目仅在后续用于验证能否稳定交付结果、增长技能和积累职业资本。

## 10. 按方向生成执行材料

Skill 不默认生成完整文章。它根据实验选择最小必要材料：

- 内容实验：首篇内容或最小招募帖。
- 访谈实验：去诱导的访谈提纲。
- 服务或咨询：简短服务说明、招募信息和首次交付清单。
- 小工具：最小问题定义、使用场景和原型范围。
- 工作坊：招募说明、最小议程和反馈问题。
- 帮助真人：任务边界、观察记录和复盘模板。

内容实验仍可输出三个选题，但只有在内容确为合适验证媒介时才生成。

## 11. 失败处理

- **资料过多**：让用户按目录、类型或时间缩小范围；不静默抽样后假装完整分析。
- **资料冲突**：并列展示证据，交给用户校正。
- **回答抽象**：追问最近一次真实事件，不继续堆叠概念。
- **证据不足**：标记低置信度并设计观察任务，不硬造优势。
- **三个方向过于相似**：至少生成一个不同角色、对象或工作方式的方向。
- **用户无法选择**：比较哪个实验最便宜、最快产生决定性信息。
- **用户要求确定答案**：重申 Skill 的输出是实验，不是命运判断。
- **实验没有反馈**：未达到预注册的有效曝光时，沉默是缺失数据；达到充分的预注册曝光后，沉默也只是对受检验响应、需求、相关性或采用组件的弱负面证据。诊断触达、招募、对象匹配、媒介和交付忠实度，不直接判定能力、能量、角色适配或整个方向失败。

## 12. Skill 文件结构

```text
README.md
LICENSE
discover-your-direction/
├── SKILL.md
├── agents/
│   └── openai.yaml
└── references/
    ├── evidence-mirror.md
    ├── interview-guide.md
    └── direction-experiment.md
```

- `SKILL.md`：触发条件、核心编排、阶段切换与强制边界。
- `evidence-mirror.md`：授权、规范的来源观察/AI 推断双记录 schema、溯源和反循环强化规则。
- `interview-guide.md`：六问、追问选择与框架校验。
- `direction-experiment.md`：方向卡、选择规则、七天实验及执行材料。
- `agents/openai.yaml`：Skill 列表所需的用户界面元数据。
- 仓库外层 `README.md` 与 `LICENSE`：GitHub 展示、安装和授权说明，不属于 Skill 运行上下文。

首版不需要脚本或资产目录。流程依赖语言推理，加入脚本不会提升可靠性。

## 13. 验证方案

至少覆盖以下场景：

1. 无资料，用户回答具体且证据充分。
2. 用户授权读取少量本地笔记。
3. AI 对话很多，但缺少一手行为证据。
4. 回答抽象、简短或频繁说“不知道”。
5. 用户资料中的不同身份互相矛盾。
6. 用户已有三个方向，无法选择。
7. 用户最有证据的能力不适合内容创作。
8. 用户只收到点赞和熟人赞美，没有真实承诺。
9. 用户对一个新身份有强动力，但过去证据很少。
10. 指定目录包含疑似敏感文件或资料规模过大。

其中至少进行两组成对测试：

- **时间压力测试**：记录核心对话耗时、被跳过的证据维度和方向卡的具体程度；判断十五分钟目标是否导致追问不足或输出泛化。
- **资料偏差测试**：让同一测试案例分别在无资料、仅 AI 对话、包含一手项目记录三种条件下运行；检查 Skill 是否把“记录频率”误判为“优势强度”，以及结论变化能否由新增证据解释。

质量标准：

- 不把推测写成事实。
- 每个方向包含反证条件。
- 方向之间存在实质差异。
- 访谈没有同时抛出多道题。
- 十五分钟阶段只选择实验，不生成冗长执行包。
- 实验匹配真实工作方式，而不是默认写内容。
- 用户能明确说出下一步行动及希望学到什么。
- 用户不必赞同 AI；能够指出误读或否定方向，也视为框架校验成功。

### 现实反馈回流

原型测试必须包含七天后的复盘，不能只评价十五分钟结束时用户是否“感觉被理解”。复盘至少检查：

- 用户是否执行了身份试演；未执行时，阻力来自方向、实验成本还是任务表述。
- 出现了哪些实质外部信号；分别是什么行为、实际投入了多少时间/金钱/声誉/隐私/努力成本、目标匹配如何、来自熟人/陌生目标对象/其他场景、出现次数及曝光分母。
- 出现了什么意外、反例或重新定义问题的时刻。
- 用户对关键假设的判断是否发生变化，以及变化依据是什么。
- 下一步是继续、调整还是停止，是否能说明理由。

若实验没有产生反馈，Skill 应先检查预注册的有效曝光是否达成，并诊断触达、招募、对象匹配、媒介、交付忠实度或实验设计。未达有效曝光时属于缺失数据；达到充分曝光后的沉默，也只是对受检验响应、需求、相关性或采用组件的弱负面证据，不能据此判断整个方向成立或失败。

## 14. Agent 可解性与人机边界

该问题属于 **B 档：可半自动化**。

Agent 可以：

- 在授权范围内整理资料、提取候选证据与冲突。
- 主持证据式访谈并提出不同解释。
- 生成方向假设、反证条件和低成本实验。
- 整理用户带回的结果并辅助复盘。

人必须：

- 决定资料授权范围并纠正 AI 对经历的解释。
- 选择愿意真实进入的角色。
- 亲自联系实践者和目标对象，发送或发布材料，招募参与者，并执行和交付现实实验。
- 提供外部反馈、内部体验及现场判断。
- 对继续、调整或停止作最终决定。

没有现实反馈回流时，Agent 最多能生成待验证解释，不能宣布已经找到合适方向。

## 15. 发布边界

首版为中文优先。仓库公开发布到 GitHub 前应完成：

- Skill 结构与 YAML 校验。
- 至少一轮无资料和一轮有资料的前向测试。
- 检查示例中不含私人资料。
- README 提供用途、安装、触发示例和隐私说明。
- 使用宽松的开源许可证，默认建议 MIT。
